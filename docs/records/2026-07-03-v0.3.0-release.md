# v0.3.0 Release Summary (Implemented + Verified)

## What Changed

- Bundles generated by `build-okf` are now OKF v0.1 conformant:
  - bundle-root `index.md` (reserved filename) is generated
  - optional `okf_version: "0.1"` is declared in root `index.md` frontmatter
- New spec-level validator: `enterprise-okf-ai okf-spec-validate`.
- Reserved filename handling now matches the spec: only `index.md` and `log.md` are reserved.
- `build-okf` overwrites the output directory each run (deterministic, no stale artifacts).
- Sample bundle regenerated; docs and notebook updated to match.

## Verification Performed

Commands run in this repo workspace:

- `make check`
  - ruff, mypy, pytest, notebook validation, and docs link checks all passed.
- `make run-e2e`
  - strict end-to-end pipeline passed with `validation_errors=0` and API smoke checks returning `200`.
- Spec conformance checks:
  - `enterprise-okf-ai okf-spec-validate --okf-dir examples/00_minimal_okf` passed
  - `enterprise-okf-ai okf-spec-validate --okf-dir examples/sample_okf_bundle` passed

## Notes for Maintainers

- Because `build-okf` now overwrites its output directory, avoid pointing it at any path that contains non-generated material you want to keep.
- If a user insists on bundle-root `README.md` for GitHub rendering, it must be treated as a normal concept document under OKF and carry frontmatter like any other concept (or be kept outside the OKF bundle root).
