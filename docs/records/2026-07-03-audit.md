# 2026-07-03 Repository Audit (OKF Zero-to-Mastery + Conformance Hardening)

This record captures what was inspected in this repository, what was found, and why changes were made.

## What Was Inspected

- Learner entrypoints: `README.md`, `docs/00-overview.md` through `docs/11-next-steps.md`, `HANDBOOK.md`.
- Examples: `examples/00_minimal_okf/`, `examples/sample_okf_bundle/`, `examples/enterprise_docs/`.
- Tooling and CLI: `src/enterprise_okf_ai/cli/main.py`, bundle generator, strict validator, retrieval/indexing/graph codepaths.
- Quality gates: `Makefile`, `scripts/check_docs_links.py`, `scripts/validate_notebook.py`, `scripts/run_real_e2e.py`, unit/integration tests.
- Release hygiene: `RELEASE_NOTES.md`, `docs/release-checklist.md`.

## Key Findings (Prioritized)

### 1) Generated bundles were not OKF v0.1 conformant

- `build-okf` produced a bundle-root `README.md` **without YAML frontmatter**.
- Multiple subsystems treated `readme.md` as “reserved” and skipped it, masking the conformance gap.
- Under OKF v0.1, reserved filenames are only `index.md` and `log.md`; all other `.md` files are concept documents and must satisfy the “frontmatter + non-empty type” bar.

Impact:
- The repo claimed “OKF-conformant bundles” while producing bundles that fail the spec’s conformance rules.
- This matters for teaching (beginners copy the shape) and for interoperability with external OKF consumers.

### 2) Strict E2E could fail due to stale output artifacts

- The generator wrote into an existing output directory without removing older files.
- After correcting “reserved filename” handling, stale `README.md` files from older runs could cause strict validation failures.

Impact:
- `make run-e2e` could regress depending on prior workspace state, which undermines “deterministic, reproducible” teaching.

### 3) Spec vs repo extensions needed one more hard boundary

- The tutorial docs were already strong, but some repo behavior contradicted the spec claims around reserved files and bundle-root index conventions.
- The notebook content leaned toward implying OKF itself is “strict” rather than “minimal spec + optional producer extensions.”

Impact:
- Beginners could walk away with the wrong mental model: “OKF requires strict enterprise schema,” which is explicitly not the spec.

## Summary: What Needed to Change

- Make generated bundles actually OKF v0.1 conformant (`index.md`, spec-accurate reserved filenames).
- Add a **spec-level** validator separate from the **enterprise strict** validator.
- Ensure bundle generation overwrites output directories to avoid stale artifact failures.
- Update docs/examples/notebook to match the corrected behavior and reinforce the two-track learning model.
