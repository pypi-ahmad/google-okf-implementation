# v0.3.0 Improvement Plan (Executed)

This is the implementation plan that was followed to ship v0.3.0.

## Goals

- Make generated bundles genuinely OKF v0.1 conformant.
- Keep a strict enterprise validator, but clearly separate it from spec conformance.
- Improve beginner correctness: no spec-vs-convention drift in docs/examples/notebook.
- Preserve and refine existing learning path rather than rewriting from scratch.

## Planned Changes

1. Bundle generation
   - Generate bundle-root `index.md` (not `README.md`).
   - Add optional `okf_version: "0.1"` frontmatter to bundle-root `index.md`.
   - Overwrite output directory to prevent stale artifacts.

2. Reserved filenames
   - Treat only `index.md` and `log.md` as reserved across the codebase.
   - Remove special-casing of `README.md` as reserved.

3. Spec conformance validator
   - Implement OKF v0.1 conformance checks (minimal 3-rule bar).
   - Add CLI command `enterprise-okf-ai okf-spec-validate`.
   - Add unit tests for both example bundles and key failure modes.

4. Docs and learning path
   - Make “Track A (OKF spec) / Track B (repo pipeline)” explicit in README + docs/00.
   - Update references that claimed repo bundles used `README.md` at bundle root.
   - Document `build-okf` overwrite semantics in the workflow doc.

5. Examples and notebook
   - Regenerate `examples/sample_okf_bundle` to match new generator behavior.
   - Update tutorial notebook text to label strict fields/relationships as repo conventions.

6. Release
   - Bump version to 0.3.0 (pyproject + API surface).
   - Update `RELEASE_NOTES.md` and `docs/release-checklist.md`.
   - Add records under `docs/records/`.
